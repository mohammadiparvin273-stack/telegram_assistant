# دستیار شخصی و منشی تلگرامی (پایتون)

یک ربات تلگرام که:
- یادآوری و کارها رو مدیریت می‌کنه (`/remind`, `/task`)
- رویداد/قرار ثبت می‌کنه (`/event`)
- اهداف یادگیری/بیزینسی‌ت رو با ددلاین پیگیری می‌کنه (`/goal`)
- هر روز یه بریف کوتاه از کارهای امروزت + یه نکته آموزشی مرتبط با تخصصت می‌فرسته
- هر هفته یه گزارش پیشرفت و پیشنهاد بهبود از هوش مصنوعی می‌ده
- یه نقشه راه شخصی‌سازی‌شده برای مهارت/بیزینست می‌سازه (`/roadmap`)
- هرچی بهش بگی به خاطر می‌سپاره و خودش وقتی زیاد شد خلاصه‌اش می‌کنه تا هوشمند بمونه
- با هوش مصنوعی رایگان گوگل (Gemini) گفتگو می‌کنه و درباره تخصصت راهنمایی می‌ده
- روی هاست رایگان (Render) به‌صورت ۲۴ ساعته قابل اجراست، با دیتابیس دائمی رایگان (Supabase)

---

## ۱. ساخت ربات تلگرام (رایگان)
1. در تلگرام به [@BotFather](https://t.me/BotFather) پیام بده.
2. دستور `/newbot` رو بزن و اسم و یوزرنیم دلخواه بده.
3. یک **توکن** بهت می‌ده، مثل: `123456:ABC-...` — این می‌شه `BOT_TOKEN`.

## ۲. گرفتن کلید رایگان هوش مصنوعی (Gemini)
1. به [Google AI Studio](https://aistudio.google.com/app/apikey) برو.
2. با اکانت گوگل وارد شو و یک API Key رایگان بساز.
3. این می‌شه `GEMINI_API_KEY`.

> توجه: نسخه رایگان Gemini محدودیت تعداد درخواست در روز/دقیقه داره؛ برای استفاده شخصی کافیه.

## ۳. اجرا روی سیستم شخصی (تست سریع)
```bash
pip install -r requirements.txt
cp .env.example .env
# مقادیر BOT_TOKEN و GEMINI_API_KEY رو داخل .env بذار
# خط WEBHOOK_URL رو خالی بذار (حالت polling فعال می‌شه)
python main.py
```

## ۴. دیپلوی رایگان و ۲۴ ساعته روی Render
1. یک ریپازیتوری روی گیت‌هاب بساز و این فایل‌ها رو داخلش push کن.
2. وارد [render.com](https://render.com) شو → New → **Web Service** → ریپازیتوری رو انتخاب کن.
3. Build Command: `pip install -r requirements.txt`
   Start Command: `python main.py`
4. Plan رو روی **Free** بذار.
5. در بخش Environment، سه متغیر بذار:
   - `BOT_TOKEN`
   - `GEMINI_API_KEY`
   - `WEBHOOK_URL` = آدرسی که Render بهت می‌ده، مثلا `https://telegram-assistant.onrender.com` (بدون `/` انتهایی)
6. Deploy کن. بعد از بالا اومدن، ربات خودش webhook رو تنظیم می‌کنه.

### نکته مهم: بیدار نگه‌داشتن سرویس رایگان
سرویس‌های رایگان Render بعد از ۱۵ دقیقه بی‌فعالیتی می‌خوابن. برای اینکه ربات واقعاً ۲۴ ساعته فعال بمونه:
1. یک اکانت رایگان در [UptimeRobot](https://uptimerobot.com) بساز.
2. یک Monitor از نوع HTTP بساز که هر ۵ دقیقه آدرس Render (`https://your-app.onrender.com`) رو پینگ کنه.
این کار سرویس رو بیدار نگه می‌داره.

### نکته مهم درباره حافظه (دیتابیس)
دیتابیس پیش‌فرض SQLite روی پلن رایگان Render **پایدار دائمی نیست**. برای حل این مشکل، پروژه الان از یک دیتابیس ابری رایگان و دائمی (Supabase / Postgres) هم پشتیبانی می‌کنه؛ کافیه مرحله بعد رو انجام بدی.

## ۵. وصل کردن دیتابیس دائمی رایگان (Supabase) — پیشنهادی
1. به [supabase.com](https://supabase.com) برو و رایگان ثبت‌نام کن.
2. یک پروژه جدید بساز (یک رمز عبور دیتابیس هم ازت می‌خواد، ذخیره‌اش کن).
3. از منوی **Project Settings → Database → Connection string** بخش **URI** رو کپی کن. چیزی شبیه این:
   ```
   postgresql://postgres:[YOUR-PASSWORD]@db.xxxxxxxxxxxx.supabase.co:5432/postgres
   ```
4. `[YOUR-PASSWORD]` رو با رمزی که ساختی جایگزین کن.
5. این مقدار رو به‌عنوان متغیر محیطی `DATABASE_URL` در Render (یا در فایل `.env` برای اجرای محلی) اضافه کن.
6. همین! دفعه بعد که ربات اجرا بشه، خودش تشخیص می‌ده `DATABASE_URL` ست شده و به‌جای SQLite از Postgres/Supabase استفاده می‌کنه. جداول لازم هم خودکار ساخته می‌شن.

با این کار، هیچ‌وقت با redeploy یا ری‌استارت شدن سرویس، یادآوری‌ها، رویدادها و حافظه پاک نمی‌شن. اگه `DATABASE_URL` رو خالی بذاری، همون SQLite ساده (بدون نیاز به هیچ سرویس اضافه) به کار خودش ادامه می‌ده — مثلا برای تست سریع روی سیستم شخصی.

## دستورات ربات
از این به بعد لازم نیست همه‌چیز رو تایپ کنی — بعد از `/start`، یه منوی دکمه‌ای پایین صفحه ظاهر می‌شه (📌 یادآوری‌ها، ✅ کارها، 🎯 اهداف، 🧠 حافظه، 📊 رشد و گزارش، ⚙️ تنظیمات) که با لمس هر کدوم زیرمنوی مربوطه باز می‌شه، و برای هر آیتم (یادآوری/کار/هدف) دکمه‌ی حذف یا انجام‌شده هم زیرش میاد. هر وقت منو گم شد، کافیه `/menu` رو بزنی تا دوباره بیاد.

اگه ترجیح می‌دی تایپ کنی، همه‌ی دستورات زیر هم دستی کار می‌کنن:
```
🗂 یادآوری و برنامه
/remind YYYY-MM-DD HH:MM متن     ثبت یادآوری
/reminders                        لیست یادآوری‌های فعال
/delreminder ID                   حذف یادآوری
/event YYYY-MM-DD HH:MM عنوان     ثبت رویداد/قرار
/events                           لیست رویدادها
/delevent ID                      حذف رویداد

✅ کارها و اهداف
/task متن                         افزودن کار به لیست
/tasks                            لیست کارهای باز
/done ID                          انجام‌شده علامت زدن
/deltask ID                       حذف کار
/goal عنوان | ددلاین | توضیح       ثبت هدف یادگیری/بیزینسی
/goals                            لیست اهداف
/delgoal ID                       حذف هدف

🧠 حافظه و رشد
/note متن                         ثبت یک نکته در حافظه
/notes                            دیدن حافظه
/forget                           پاک کردن کل حافظه
/roadmap                          نقشه راه شخصی‌سازی‌شده
/weekly                           گزارش هفتگی فوری

⚙️ تنظیمات
/dailytime ساعت(0-23)             ساعت ارسال بریف روزانه
/weeklyday روز(0=دوشنبه..6=یکشنبه) روز ارسال گزارش هفتگی
/menu                             نمایش دوباره دکمه‌ها
/help                             راهنما
```
همچنین می‌تونی به زبان طبیعی هم بنویسی، مثلا: «فردا ساعت ۹ صبح یادم بنداز با علی تماس بگیرم».

### بریف روزانه و گزارش هفتگی خودکار
از همون اول ساعت پیش‌فرض بریف روزانه ۸ صبح و روز گزارش هفتگی جمعه (weekly_day=4) هست. با `/dailytime` و `/weeklyday` می‌تونی عوضشون کنی. این‌ها بر اساس ساعت سروری که ربات روش اجرا می‌شه حساب می‌شن (نه ساعت محلی خودت)، پس اگه با ساعت واقعی‌ت فرق داشت با `/dailytime` تنظیمش کن.

## ساختار پروژه
```
main.py         نقطه ورود، اتصال هندلرها و اجرای webhook/polling
config.py       خواندن متغیرهای محیطی
database.py     واسط انتخاب دیتابیس (بسته به DATABASE_URL بین SQLite و Postgres سوییچ می‌کنه)
db_sqlite.py    لایه دیتابیس SQLite (پیش‌فرض، محلی)
db_postgres.py  لایه دیتابیس Postgres/Supabase (دائمی، پیشنهادی برای دیپلوی)
ai.py           اتصال به Gemini برای گفتگو و راهنمایی تخصصی
keyboards.py    ساخت منوهای دکمه‌ای (ReplyKeyboard و InlineKeyboard)
scheduler.py    بررسی دوره‌ای و ارسال یادآوری‌های سررسید شده
handlers.py     تمام دستورات و منطق مکالمه
```

## توسعه‌های پیشنهادی بعدی
- افزودن دکمه‌های شیشه‌ای (Inline Keyboard) برای مدیریت راحت‌تر کارها و یادآوری‌ها
- اتصال به تقویم گوگل برای همگام‌سازی قرارها
- پشتیبانی از تایم‌زون شخصی هر کاربر برای بریف روزانه/هفتگی
- امکان اتصال چند هدف به هم و پیگیری درصد پیشرفت هر هدف
